# Ultracite Rules Integration

При работе с файлом `ultracite.mdc` следуй этим принципам:

## Структура файла
- Файл содержит правила форматирования и линтинга для TypeScript/JavaScript проектов
- Использует YAML front matter для метаданных
- Организован по категориям: Accessibility, Code Quality, React/JSX, TypeScript и т.д.

## Принципы редактирования
1. **Сохраняй структуру**: Не изменяй основную организацию по категориям
2. **Добавляй контекст**: Каждое новое правило должно иметь краткое объяснение
3. **Примеры кода**: Включай примеры "хорошего" и "плохого" кода где это уместно
4. **Консистентность**: Используй единый стиль описания правил

## Категории правил
- **Accessibility (a11y)**: Правила доступности для веб-приложений
- **Code Complexity**: Правила сложности и качества кода
- **React/JSX**: Специфичные правила для React компонентов
- **TypeScript**: Правила для TypeScript кода
- **Style**: Правила стиля и консистентности
- **Testing**: Правила для тестов

## Формат правил
Каждое правило должно:
- Начинаться с "Don't" или "Use" для ясности
- Быть конкретным и actionable
- Включать контекст когда это необходимо
